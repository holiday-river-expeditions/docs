### Design Decisions

- **Stardos Stencil reserved for H1 hero moments only** — prevents overexposure, keeps impact high
- **Canyon (#1a1612) as primary dark background** — warmer than charcoal, evokes canyon walls at dusk
- **Teal as accent thread** — top-borders on cards, CTA buttons, separator dots, star ratings — creates subliminal water connection
- **Section rhythm** alternates: hero → off-white → canyon-dark → sand → white → canyon-dark → full-bleed photo
- **Placeholder images from Unsplash** — will be replaced with Holiday's photography once Sanity content is wired up

## Session 3: Sanity CMS Wiring

### Accomplished

- Created `homepage` singleton schema — hero (heading, subheading, image, CTA), "The Holiday Way" (tagline, heading, body), final CTA (heading, subheading, image, button)
- Added `featured` boolean to `trip` schema — allows Holiday to mark trips for the homepage "Signature Trips" section
- Configured Sanity Studio structure — `homepage` and `siteSettings` appear as singletons at the top of the Studio sidebar with a divider, other document types below
- Added GROQ queries: `homepageQuery` and `featuredTripsQuery`
- Added fetch helpers: `getHomepage()` and `getFeaturedTrips()`
- Regenerated Sanity types (10 queries, 25 schema types)
- Rewrote homepage as async server component that fetches from Sanity:
  - Hero, "The Holiday Way", and final CTA pull from the `homepage` singleton
  - River cards pull from `river` documents via `getAllRivers()`
  - Trip cards pull from `trip` documents via `getFeaturedTrips()`
  - All images use `urlFor()` for Sanity CDN optimization
  - Every field has a hardcoded fallback — page works before and after Studio content is added
- Fixed `@sanity/image-url` deprecation — switched to named `createImageUrlBuilder`
- Replaced Unsplash placeholders with actual bikeraft.com photography
- Added bikeraft.com domains to Next.js `remotePatterns`
- All checks pass: typecheck, lint, production build (clean, no warnings)

### Decisions

- **Homepage is a Sanity singleton** — purpose-built editing experience, not a generic page
- **Testimonials stay hardcoded** — long-term plan is third-party review integration (TripAdvisor, Google, Yelp)
- **Trust bar stays hardcoded** — changes rarely, singleton is overkill for 4 strings
- **Fallback pattern** — every Sanity field has a `?? fallback` default
